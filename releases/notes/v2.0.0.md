## Marketplace

- Velaris App Store inside Studio — browse, sandbox, and install platform extensions.
- Two tiers in one repo (`Velaris-App-OS/Marketplace`): **official/** (Velaris-published)
  and **community/** (fork + PR).
- **Unspoofable Official tier:** `_effective_tier` grants Official only when the
  source org is `velaris-app-os`, the source is under the `official/` folder, **and**
  the package id is in the in-image allowlist `official_registry.json`. A manifest
  never decides its own tier.
- Install pipeline: download → `verify_checksum` → `parse_and_validate_manifest`
  before anything is enabled; per-tenant install/uninstall (revoke, and
  revoke + delete-data teardown).
- Out-of-process trust model — third-party packages never run in-process platform code.

## Test Suite (core) + HxTest (marketplace AI layer)

- **Test Suite** — always-on: deterministic Run / History / Conformance, a closed
  DSL + executor + runner, ephemeral-tenant isolation, and built-in smoke/component/
  security suites.
- **Conformance gate** — workspace submission is hard-blocked unless structural
  conformance passes.
- **Structural test generation** is core (`/testsuite/generate`).
- **HxTest** (install-gated add-on) — AI scenario generation via HxNexus, live
  regeneration, rationale, and the Studio Generated tab; advisory only, degrades
  cleanly when AI is unavailable.

## HxVault — per-tenant encryption

- Envelope encryption: a random per-tenant DEK wrapped by a master KEK
  (`VELARIS_CASE_KEK`); platform DEK for tenant-less data.
- Self-describing `hxv2` ciphertext with AAD; back-compatible with `hxv1`.
- Multi-worker cache reconciliation (periodic resync) so DEK create/shred is
  consistent across uvicorn workers.

## HxDB Manager + Security

- AI-assisted database operations panel inside Studio.
- Security layer: credential masking, breach detection, account auto-disable,
  per-user execute rate limiting, and DML before-image logging.

## Customer Accounts

- Persistent portal identity — OTP login, case history across visits, and
  GDPR erasure.

## Security & architecture hardening

- API gateway: single stable `/api/v1` door (nginx), case-service loopback-only.
- `safe_expression` replaces `eval`-based rule evaluation (RCE removed).
- HxGuard: 404 anti-oracle responses and filtered listings.
- Universal LLM choke point (`GuardedBackend`): prompt-size cap, prompt-injection
  → security event, and output secret-scrubbing on every model call.
- Temporal durable-timer companion for SLA durability.
- Passkeys / WebAuthn.

## Infrastructure & technology changes

This release trims and modernizes the runtime stack — review before upgrading:

| Component |   v1.0.0   | v2.0.0                                       | Action on upgrade |
|-----------|------------|----------------------------------------------|-------------------|
| Cache     | Redis      | **Valkey** (`valkey/valkey:8-alpine`)        | Old `redis` container + image can be removed |
| Search    | OpenSearch | **Removed** — zero platform references       | Stop & remove the `opensearch` container and `search-data` volume |
| Auth      | Keycloak   | **Removed** — login uses the built-in stack  | Stop & remove the `keycloak` container |
| Secrets   | `.env`file | **OpenBao** (Group K) — `.env` is rendered from the vault, fail-closed | Opt in via `deploy/openbao/enabled`; manage with the `scripts/secrets-*.sh` tooling |

Docker Compose secrets were also hardened. The updated `uninstall-velaris.sh` reflects
the new stack (Valkey + OpenBao, OpenSearch/Keycloak dropped) and shreds OpenBao secrets.

## Upgrade from v1.0.0

1. Apply new migrations in `migrations/` (through `093_testsuite_ai_stale.sql`),
   including HxVault (`090`), Test Suite (`091`), marketplace conformance (`092`).
2. Set `VELARIS_CASE_KEK` (HxVault master key) — `setup-velaris.sh` generates one.
3. Place the updated `releases.txt` on the server and let `start-velaris.sh` apply
   `releases/manifest.sql` (activates the four feature flags above).
4. Activate the marketplace (creates its tables) to make HxTest installable.
5. Retire the dropped infrastructure: remove the old `redis`, `opensearch`, and
   `keycloak` containers/images and the `search-data` volume (see the table above).
   `start-velaris.sh` brings up Valkey and (if enabled) OpenBao automatically.

## Installation (fresh)

```bash
./setup-velaris.sh
```
